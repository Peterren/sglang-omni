# Audar benchmark artifacts

These are machine-generated, lightweight verification artifacts for the locked
comparison in `../RESULTS.md`.

- `performance/`: four seven-request H100 runner outputs, the paired driver's
  exact `performance_summary.json`, focused pytest logs, and the WAV hash
  manifest connecting the new runs to the earlier quality-evaluated files.
- `quality/`: direct Arabic ASR outputs and `quality_summary.json` generated by
  `../summarize_quality.py`. The generation files contain the pinned FLEURS
  selection and per-sample hashes. English translation is not used.

WAV files are excluded. Every paired run produced PCM WAV SHA-256
`e7f0b6bb3ea5d950ff2fc1329a63954c2579cf97201dafb0a61514eb9b5ca04b`.
The raw WAVs remain staged at
`/data/jaxan/audar-results-production-equal-final` and
`/data/jaxan/audar-quality-results-fleurs50-nodigits` on H100 pending upload
with the correct Hugging Face owner credential.
